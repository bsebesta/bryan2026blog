{{- /* Reusable segmented control — pure CSS, no JS. Renders a group of sr-only radios plus their labelled buttons. The caller supplies a sibling `
` whose Nth `
` pairs with the Nth option. The show/hide and active-button styling live in baseof (@layer base) and are keyed by POSITION, not id — so this control is reusable with any labels or views, here or elsewhere. Takes a dict: name — radio group name; MUST be unique per page (also seeds the ids) options — slice of dicts: { label, value?, checked? } aria — accessible group label (default "Choose a view") Mark one option `checked: true`; if none is, the first is selected. Usage:
{{ partial "segmented.html" (dict "name" "view" "options" (slice (dict "label" "Cover" "checked" true) (dict "label" "Cards") (dict "label" "Thumbnail"))) }}
…first view…
…second view…
…third view…
Keep the radios, .seg, and .seg-views as siblings — the CSS uses the general sibling combinator, which only reaches elements at the same level. Supports up to six options (extend the nth-of-type rules in baseof for more). */ -}} {{- $name := .name -}} {{- $opts := .options -}} {{- $aria := .aria | default "Choose a view" -}} {{- $anyChecked := false -}} {{- range $opts }}{{ if .checked }}{{ $anyChecked = true }}{{ end }}{{ end -}} {{- range $i, $o := $opts -}} {{- $slug := (or $o.value (printf "%d" $i)) | anchorize -}} {{- $id := printf "%s-%s" $name $slug -}} {{- $checked := $o.checked -}} {{- if and (not $anyChecked) (eq $i 0) }}{{ $checked = true }}{{ end -}} {{- end }}
